# 深さチェック（Depth Check）

以下の観点でドラフトを評価し、浅い主張を発見・修正する。

## チェック項目

### 1. 「それ、Googleで出てこない？」テスト
- その段落の主張をそのまま検索したら、似た記事が大量に出てくるか？
- 出てくるなら、著者独自の切り口・経験・具体例が不足している
- **対処**: 著者の実体験、独自の観察、具体的な数字・エピソードに置き換える

### 2. 「なぜ？」3回深掘り
- 主張に対して「なぜそう言えるのか？」を3回繰り返す
- 3回目の「なぜ」に答えられないなら、その主張は根拠が浅い
- **対処**: 因果関係を明示する。「AだからB」ではなく「Aのとき、Cという条件下でBが起きやすい。なぜなら〜」

### 3. 具体 vs 抽象のバランス
- 抽象的な主張が2文以上続いていないか？
- 具体例なしの段落がないか？
- **対処**: 抽象→具体→抽象のサンドイッチ構造にする。主張したら必ず具体例を挟む

### 4. 「で、どうすればいい？」テスト
- 読者が読み終わった後、明日から何を変えられるか明確か？
- 「大事です」「必要です」で終わる段落は、So What？が欠けている
- **対処**: 具体的なアクション、判断基準、チェックリストを提示する

### 5. 反論耐性チェック
- 「でも〜の場合は？」「それは〜だからでは？」と突っ込まれる余地はないか？
- 想定される反論を先回りして潰しているか？
- **対処**: 「もちろん〜という反論もある。しかし〜」の構文で対処する。ただし、すべての反論に対処する必要はなく、致命的なものだけ扱う

## 安直な主張のパターン（検出したら即修正）

| 安直なパターン | 修正の方向性 |
|---|---|
| 「〜は大事/重要」 | なぜ大事だと分かっていても実践できないのか？を書く |
| 「〜すべき」 | すべきだと分かっているのにできない構造的理由を掘る |
| 「〜が増えている」 | 増えた結果、何が起きているか？誰が困っているか？を書く |
| 「〜と〜のバランスが大切」 | バランスとは具体的に何対何か？どう測るか？を書く |
| 「人それぞれ」 | それでも共通する構造や傾向はないか？を探る |
