# 文体チェック（Style Check）

以下の観点でドラフトを評価し、著者らしい文体を維持しつつAI臭さを排除する。

## チェック項目

### 1. AI特有の表現の排除
以下の表現が含まれていたら、即座に書き換える：

| AI的な表現 | 問題点 | 修正の方向性 |
|---|---|---|
| 「〜と言えるでしょう」 | 断定回避の逃げ | 言い切るか、具体的な条件付きで述べる |
| 「〜ではないでしょうか」 | 読者への同意強要 | 主張として書き、根拠を添える |
| 「〜において」「〜における」 | 硬い書き言葉 | 「〜で」「〜の」に置き換える |
| 「〜することが重要です」 | 教科書的で無個性 | なぜ重要か、やらないとどうなるかを書く |
| 「さまざまな」「多くの」 | 具体性の欠如 | 具体的な数や例を挙げる |
| 「〜という側面があります」 | 曖昧な表現 | 「〜だ」と直接述べる |
| 「まず〜、次に〜、最後に〜」 | 機械的な列挙 | 自然な接続に書き換える |
| 「いかがでしたか？」 | 典型的なAI/量産記事の締め | 削除し、本質的なまとめにする |
| 「〜についてご紹介しました」 | 紹介者ポジション | 著者の主張として締める |

### 2. 著者の声の再現
- `knowledges/` の素材から学んだ著者の口調を維持しているか？
- 著者が使いそうにない表現が混ざっていないか？
- **対処**: 素材を再読し、語彙・文末表現・比喩のパターンを確認する

### 3. 文末表現の多様性
- 「〜です」「〜ます」が3回以上連続していないか？
- 「〜だ」「〜である」が単調に続いていないか？
- **対処**: 体言止め、問いかけ、引用、会話調などを織り交ぜる

### 4. 一文の長さ
- 80文字を超える文がないか？（読みにくさの原因）
- 主語と述語が離れすぎていないか？
- **対処**: 一文一意を原則とし、長い文は分割する

### 5. 不要な修飾語の削除
- 「非常に」「とても」「本当に」「まさに」が多用されていないか？
- 削っても意味が変わらない修飾語はないか？
- **対処**: 修飾語を削除し、具体的な描写や数字で強調する

### 6. 読者との距離感
- 上から目線になっていないか？（「〜すべきです」の連発）
- 逆に、へりくだりすぎていないか？（「〜かもしれません」の連発）
- **対処**: 対等な立場で語りかける。共に考える姿勢を示す

## 最終チェック：音読テスト

頭の中で音読し、以下を確認する：
- 口に出して自然に読めるか？
- つっかえるポイントはないか？
- リズムが心地よいか？

音読してひっかかる箇所は、読者も引っかかる。滑らかに読める文章に整える。
