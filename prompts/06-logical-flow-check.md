# 流れチェック（Logical Flow Check）

以下の観点でドラフトを評価し、論理の飛躍や読者の離脱ポイントを発見・修正する。

## チェック項目

### 1. 段落間の接続チェック
- 各段落の最後の文と、次の段落の最初の文を並べて読む
- 「なぜこの話の次にこの話？」と感じたら、接続が切れている
- **対処**: 接続詞に頼るのではなく、前の段落の結論が次の段落の前提になる構造にする

### 2. 論理の飛躍チェック
- 「A → B → C」の流れで、Bが省略されていないか？
- 著者には自明でも、読者には飛躍に見える箇所がないか？
- **対処**: 省略されたステップを補う。ただし冗長にならないよう、1文で橋渡しする

### 3. 読者の前提知識チェック
- 専門用語や固有名詞を説明なしに使っていないか？
- 読者が「それ何？」と立ち止まるポイントはないか？
- **対処**: 初出時に簡潔な説明を添える。ただし、ターゲット読者が明らかに知っている用語は説明不要

### 4. 冒頭と結末の整合性
- 冒頭で提示した問い・テーマに、結末で答えているか？
- 途中で論点がすり替わっていないか？
- **対処**: 冒頭の問いを明確にし、結末でその問いに直接答える形にする

### 5. 情報の出し順チェック
- 読者が必要とする情報が、必要なタイミングで出ているか？
- 結論を理解するための前提が、結論の後に出ていないか？
- **対処**: 「読者がここで知りたいことは何か？」を各段落で自問し、順序を入れ替える

### 6. テンポとリズム
- 長い段落が3つ以上続いていないか？（読者の集中力が切れる）
- 同じ構文パターンが繰り返されていないか？（単調になる）
- **対処**: 長短を混ぜる。問いかけ、短い断言、具体例の挿入でリズムを変える

## 構造パターンの確認

文章全体が以下のいずれかの構造に沿っているか確認する：

| パターン | 構造 | 適するケース |
|---|---|---|
| 問題提起型 | 問い → 分析 → 主張 → 具体例 → まとめ | 論考、意見記事 |
| ストーリー型 | 導入 → 展開 → 転換 → 結論 | エッセイ、体験記 |
| リスト型 | 概要 → 項目1 → 項目2 → … → 総括 | ハウツー、Tips系 |
| 対比型 | A案 → B案 → 比較 → 結論 | 選択・判断系 |

構造が曖昧な場合、読者は「この文章は何を言いたいのか」を見失う。どのパターンかを明確にし、逸脱している部分を修正する。
